# 安卓虚拟机接入与使用指南

> 本指南面向**使用者**：如何在平台上创建并使用 Android 虚拟手机（基于 Android 官方 Emulator）。
> 虚拟手机启动后会作为一台普通 Android 设备进入设备池，**和真机走同一条链路**——被调度、进工作台、执行任务、出报告。

---

## 1. 虚拟手机是什么

- 它是跑在 **Agent 宿主机**上的 **Android 官方 Emulator**。启动成功后，自动作为一台 `android` 设备出现在「设备总览」，可被调度执行任务、可进工作台手动操作。
- 价值：**便宜、可批量、可快速拉起**的 Android 执行能力——当真机不够用时，用虚拟手机弹性补充执行单元。

---

## 2. 能力边界（先读这一节，避免误用）

**虚拟手机复刻的是「规格」，不是「机型」。** 它跑的是通用 Android 系统镜像，不是某款手机的厂商系统。

**适合用虚拟手机：**
- 不同**屏幕分辨率 / dpi**、不同 **Android 版本**、不同**性能档（内存/存储）**的兼容性验证；
- UI 适配、通用业务流程跑通、批量回归、冒烟测试；
- 需要大量并发执行单元、对"是不是某款真机"不敏感的场景。

**必须用真机（虚拟手机做不到）：**
- 依赖厂商 ROM 行为（MIUI / ColorOS / 鸿蒙等定制系统、推送、特有权限弹窗）；
- 设备指纹 / 真伪检测、风控；
- 相机 / 传感器 / GPU 的真机表现、性能功耗实测。

**由此带来的两个正常现象（不是 Bug）：**
- 进系统看「关于本机」，显示的是通用模拟器型号（如 `google sdk_gphone64_arm64`），**不是你选的机型**。
- 选了不同机型、但规格（分辨率/版本/dpi/内存）相同的两台虚拟手机，**行为是等价的**。选机型本质是"选一套贴近真机的规格 + 市场参考"，**不会让它变成那台真机**。

---

## 3. 设备库与 CSV（设备模板从哪来）

创建虚拟手机时选的"设备模板"，来自**官方设备目录 CSV 导入**，不是手写的。

### 3.1 两类数据来源

- **Google Play Console 设备目录 CSV**：规格齐全（屏幕、内存、芯片、ABI、支持的 Android 版本），**导入后即可直接选用**。
- **Google 公开 `supported_devices.csv`**：只有设备身份（品牌 / 型号），没有规格，**只能作为候选**，补全规格前不可直接选。

### 3.2 CSV 需要哪些列（Play 目录格式）

导入的 CSV 建议包含这些列（大小写/空格不敏感）：

```text
Manufacturer、Brand、Model Name、Model Code、Device、Form Factor、
Screen Sizes（分辨率）、Screen Densities（dpi）、RAM (TotalMem)、
System on Chip、GPU、ABIs、Android SDK Versions（支持的安卓版本）、OpenGL ES Version
```

其中：
- **Screen Sizes** → 解析成屏幕宽高（虚拟机的分辨率）；
- **Android SDK Versions** → 这台机型支持的安卓版本区间（设备卡片上的「Android 13-15」就来自这里）；
- **ABIs** → 决定能在什么 CPU 架构的 Agent 上跑。

### 3.3 导入后系统自动做的处理

- 只保留**手机 + 平板**（智能手表 / TV / 车机等过滤掉）；
- 品牌名归一、按分辨率分档、去重；
- 列表只展示**可建机型**：**Android 5.0（API 21）及以上**——这是 Android Emulator 的下限。

### 3.4 怎么导入

- **页面**：「虚拟机」页左上角「导入设备库 CSV」上传（覆盖式刷新设备库）。
- **接口**：见《对外调用清单》(`external-api`) 中的设备库导入接口。

## 4. 怎么创建一台虚拟手机

在「虚拟机」页，按下面流程：

```text
1) 筛选：按 设备类型 / 品牌 / 系统版本 / 分辨率 / 内存 多维筛选，
   也可用搜索框按 型号 / 品牌 / device 代号 精准查找（筛选项带数量提示）。
2) 选机型：卡片显示 品牌·型号·分辨率·dpi·内存，以及该机型支持的「Android X-Y」版本区间。
3) 真机画像锁定：选了真机机型后，系统版本/屏幕/分辨率等「出厂画像」会被锁定（保证贴近真机），
   只开放「执行资源」(内存/存储) 让你按宿主机算力下调。
4) 设备别名：必填、需唯一——它代表这台设备、用于分辨和调度，创建后可随时改。
5) 创建配置：创建与下发分离，先存配置、不立即开机。
6) 下发：对配置点「探查」，系统实时询问在线 Agent 的能力（是否装好对应系统镜像），
   你从可用 Agent 里选一台下发。
7) 启动：Agent 自动建机、开机；开机完成且就绪后，进入「设备总览」，即可调度/进工作台。
```

---

## 5. 别名与调度（重要）

- 别名是**调度入口**：外部投递任务时，`deviceAliasPools` 里写这个别名，就能**精确指定派到这台虚拟手机**——和真机完全一致。
- **注意**：虚拟手机要**至少启动过一次**，它的别名才进入调度名册。一台刚创建、从没启动过的虚拟手机，直接用别名投递会被拒（提示别名不存在）。正常用法是"先启动、再派任务"。

---

## 6. 宿主环境要求（在哪台机器跑）

虚拟手机跑在 **Agent 宿主机**上，该机器需要装好：

- **JDK 17+**、**Android SDK 命令行工具**、**Emulator**、以及对应的 **system image**；
- system image 的架构要和宿主 CPU 匹配：
  - Apple Silicon（M 系列 Mac）→ `arm64-v8a`
  - Intel / AMD / **Windows** → `x86_64`
- **跨平台**：macOS / Linux / Windows 都支持。工具发现靠环境变量（`ANDROID_HOME` / `ANDROID_SDK_ROOT`）+ PATH 自动完成，**装在哪个盘、什么用户名都行**，不依赖固定目录。
- 详细安装步骤见《Agent 接入部署指南》(`agent-deployment`)；Windows 见其中的 Windows 专章。

## 7. 常见问题

- **「关于本机」不是我选的机型？** 正常。虚拟手机不复刻机型/ROM，见 §2。
- **设备总览看不到刚建的虚拟手机？** 配置创建后要「下发 → 启动」，开机并就绪后才会出现在设备总览。
- **重启 Agent / 服务后虚拟手机会丢吗？** 不会。系统会自动重新认领恢复——还在跑的恢复为运行中，已停的保留为已停止。
- **虚拟手机里没有我要测的 App？** 虚拟手机是干净的原生系统，需要的 App 要自行安装。
- **想要更真实的机型行为？** 那超出虚拟手机能力范围，请用真机（见 §2 边界）。
- **选 2GB 内存的低配机型起不来？** 低内存 + 新安卓版本可能开机失败；这是真实约束，调高内存或换更合适的版本即可。
