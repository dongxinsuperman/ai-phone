## 1. 先一眼看懂架构

```
┌────────┐      WS/HTTP       ┌─────────┐     USB     ┌──────────┐
│ 前端 F │ ────────────────→ │ Server D│                │ iPhone   │
└────────┘                    └────┬────┘                │/Android  │
                                   │ WS                  └────┬─────┘
                                   ↓                          │
                               ┌───────┐   usbmuxd/adb/pmd3   │
                               │Agent E│ ─────────────────────┘
                               └───────┘
                        （iOS 还要 A: tunneld 做 DVT 截图 = 默认方案）
```

- **D/E/F = 必开**（Server / Agent / 前端），Android 就这仨搞定
- **A = iOS 专属**（`pmd3 tunneld`，默认镜像方案要它）
- 以前还要 B（Xcode WDA）、C（iproxy），现在 agent 内置 `xcodebuild test` + usbmux 转发，**只要 `.env` 配好 `AI_PHONE_WDA_PROJECT_DIR`，B/C 都省了**

## 3. iOS 首次接入（一次性）

> **"首次"是指这台 iPhone **从来没**在这台 Mac 上装过开发者证书和 WDA**。如果以前装过，跳到 §4 即可。

下面 5 步全是**一次性**的，全部做完以后这台 iPhone 在这台 Mac 上就算"认识"了，下次插上只需要走 §4 日常流程。

```
步骤流（做完就忘，别回来了）：

  1) Mac & iPhone 准备   →   2) iPhone 解锁 + 信任这台 Mac
        ↓
  3) iPhone 打开"开发者模式"  →   4) Mac 上挂 DDI（开发者磁盘镜像）
        ↓
  5) Xcode 打开 WDA 工程，改 3 个东西，按 Cmd+U 跑一次
        ↓
     iPhone 弹"未受信任开发者"→ 去「设置→VPN 与设备管理」点信任
        ↓
     屏幕上出现 "Automation Running" 灰屏 = 大功告成
```

### 3.1 Mac & iPhone 准备

- **Mac**：装好 Xcode（已验证 26.0.1）、Xcode Command Line Tools、一个免费 Apple ID
- **iPhone**：系统设置里登了一个 Apple ID（可以和 Xcode 上同一个，用免费 ID 就行）
- **线**：用**原装或数据传输 MFi 认证线**。充电线里不少只通电，连 USB 数据都握不上手，agent 日志会刷 "udid 不在 usbmux 设备列表里"

### 3.2 iPhone 解锁 + 信任这台 Mac + 关掉自动锁屏

1. 用数据线把 iPhone 插到 Mac
2. iPhone 亮屏解锁（**第一次必须亮屏，不然弹窗出不来**）
3. iPhone 弹 "要信任此电脑吗？" → **点 Trust / 信任** → 输锁屏密码
4. **把自动锁屏关掉**：iPhone 设置 → 显示与亮度 → 自动锁定 → **永不**

   > 为什么是硬性要求：iOS 18 / 26 起，设备锁屏时连最基本的 `lockdown StartSession` 都会返回 `PasswordProtected`——**即使 pair record 里有 EscrowBag 也绕不过，是 iOS 本身的限制**。  
   > agent 每 5 秒 rescan 一次，iPhone 默认 30 秒就自动锁屏，一旦锁屏 agent 就读不到元信息，web 上设备卡片会进入「锁屏中」橙色状态，继续操作全部挂起。  
   > **对策：把自动锁定设为「永不」，或至少 15 分钟 + 工作时保持点亮。**

5. Mac 上验证一下认到了：

   ```bash
   cd /Users/dongxin/代码文件/sonic合集/ai-phone/backend
   source .venv/bin/activate
   pymobiledevice3 usbmux list
   ```

   看到 iPhone 的 UDID + 名字就行。**没看到：拔下重插 + 重跑一次 Trust 弹窗**。

### 3.3 iPhone 打开开发者模式（iOS 16+ 必须）

1. iPhone 上：设置 → 隐私与安全性 → 往下翻到「**开发者模式**」→ 打开
2. iPhone 会提示重启 → 重启
3. 重启后再亮屏 → 设置里重新进「开发者模式」→ 按提示「打开」+ 输密码

> 看不到「开发者模式」菜单？先做 3.5（用 Xcode 跑过一次 WDA 试运行），iOS 会自动把这个菜单显出来。然后回来把开关打开。

### 3.4 Mac 上挂 DDI（开发者磁盘镜像）

DDI = "iOS 开发者磁盘镜像"，iOS 17+ 的截图/调试服务依赖它。**只要没挂、agent 就起不来**。

```bash
cd /Users/dongxin/代码文件/sonic合集/ai-phone/backend
sudo -E .venv/bin/python -m pymobiledevice3 mounter auto-mount --udid <你的UDID>
```

看到下面任一条就算成功：

- `INFO DeveloperDiskImage mounted successfully` ✅
- `ERROR DeveloperDiskImage already mounted` ✅（已经挂过，等同成功）

> **这条命令做完就关，不用留着。** 每次 iPhone 重启需要重做一次（见 §5）。

### 3.5 Xcode 打开 WDA 工程，改 3 个东西，跑一次

WebDriverAgent 工程**已 vendored 在项目内** `ai-phone/third_party/WebDriverAgent/`，直接打开 `third_party/WebDriverAgent/WebDriverAgent.xcodeproj` 即可，不需要去 GitHub 单独 clone。

用 Xcode 打开 `.xcodeproj`，按下面顺序改：

1. 左侧 TARGETS 选 **`WebDriverAgentRunner`**
   - **Signing & Capabilities** → **Team** 选你的 Apple ID（免费 Team 也行）
   - **Bundle Identifier** 改一个**全世界唯一**的（建议 `com.<你名字拼音>.wda`）。免费 Apple ID 对同一个 bundle id 每年只能签 10 个，改唯一的避免和别人冲突
2. 同一页，重复 1 给 **`IntegrationApp`** target 也设一次 Team + 唯一 Bundle Id
3. 左上选目标设备为你的真机（插好的 iPhone）
4. 顶部菜单 **Product → Test**（快捷键 `Cmd + U`），**不是 Run**

然后：

- **Xcode 会编译 3-5 分钟**（第一次冷编译），编译完会往 iPhone 装 2 个 app（WebDriverAgentRunner / IntegrationApp）
- iPhone 上可能会弹 "**未受信任的开发者**"
  - → 点掉 → **设置 → 通用 → VPN 与设备管理** → 找到你的 Apple ID → 点「**信任**」
  - → 回到 Xcode 再按 `Cmd + U`
- iPhone 屏幕上出现灰底白字 **`Automation Running`** = **成功**
- Xcode 底部一直显示 "Test Running"，别关 Xcode / 别关 iPhone

如果你不想留 Xcode 窗口，按 `Cmd + .` 停掉这次 Test 没关系——**首次跑通就已经完成"代码签名 + 证书安装"了**，以后靠 §4 的 agent 自动化拉起就行。

### 3.6 把 `.env` 里的路径配上（一次性）

打开 `ai-phone/backend/.env`，把 WDA 工程路径换成你自己的：

```bash
AI_PHONE_WDA_PROJECT_DIR=/Users/<你>/<clone位置>/ai-phone/third_party/WebDriverAgent
AI_PHONE_WDA_SCHEME=WebDriverAgentRunner-nodebug   # 可选，默认就是这个
AI_PHONE_WDA_BUNDLE_ID=com.<你>.wda                # 建议唯一值
AI_PHONE_WDA_TEAM_ID=<你的 Apple Team ID>           # 10 字符大写
```

> 配好后 agent 以后就能自己跑 `xcodebuild test -allowProvisioningUpdates`，你再也不用手动开 Xcode Cmd+U。

**做到这里 §3 结束。下一次插上 iPhone → 直接跳 §4。**

## 4. iOS 日常（4 个终端）

假设：§3 做过了、`.env` 里 `AI_PHONE_WDA_PROJECT_DIR` 配好了、iPhone 已插入并解锁。

> ### ⚠️ 插上 iPhone 之后、启动 Agent 之前：**必须先点亮 iPhone 屏幕并解锁（面容/密码）**
>
> 锁屏状态下 `xcodebuild test` 会卡在
> `Run Destination Preflight: The destination is not ready.` /
> `Error Domain=com.apple.dt.deviceprep Code=-3 "Unlock None to Continue"`，
> **表象 = 终端 E 一直刷 "WDA 还没就绪"、iPhone 上始终没出现 "Automation Running"**。
>
> **做法**：先亮屏 → 解锁 → **在 Agent 把 WDA 拉起来之前的 30-120 秒内别让屏幕自动熄灭**
> （建议临时把「设置 → 显示与亮度 → 自动锁定」调到 5 分钟或「永不」）。
>
> 一旦 iPhone 上出现 "Automation Running" 灰屏，WDA 已经驻留，之后再熄屏也无所谓。

按顺序开这 4 个终端：

```
┌─────────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐
│ A tunneld   │ →│ D Server│ →│ E Agent │ →│ F 前端  │
│（sudo，常驻）│   └─────────┘   └─────────┘   └─────────┘
└─────────────┘
   iOS 专属        Android/iOS 共用
```

每个终端的具体命令看 [§9 命令参考](#9-命令参考)。

**为什么不用 iproxy / Xcode 手动 Cmd+U**：agent 启动时自己会跑 `xcodebuild test -allowProvisioningUpdates` 把 WDA 拉起来，又会用 usbmuxd 把 `127.0.0.1:8100` 打通到 iPhone。你啥都不用做。

**agent 退出时会自动停掉那个 xcodebuild 子进程**，iPhone 上的 `Automation Running` 会消失。

## 4.5 HarmonyOS 日常（3 个终端）

> **首次准备**（一次性，每台 Mac × 每台手机各做一次）：装 `hdc` + `pip install -e ".[harmony]"` + 设备开发者模式 → 详见 [`HarmonyOS环境配置笔记.md`](./HarmonyOS环境配置笔记.md)。**做完一次以后回到这里。**

启动顺序和 Android 完全一样，**不需要 tunneld**（DVT 是 iOS 专属，鸿蒙走 hdc + uitest socket）：

```
┌─────────┐    ┌─────────┐    ┌─────────┐
│ D Server│ →  │ E Agent │ →  │ F 前端  │
└─────────┘    └─────────┘    └─────────┘
```

具体命令同 §2 / §9 命令参考，agent 启动后 5 秒内自动通过 `hdc list targets` 把鸿蒙机扫进来。Web 首页设备卡 `platform` 字段会显示 `harmony`。

**和 iOS / Android 共存**：三端不抢锁、不抢端口、不抢驱动；插哪个都按各自链路接进来。

### 平时操作要点

- 第一次插上手机会弹"允许 USB 调试" → 勾"一律信任" → 确认（和 Android 体验一致）
- 锁屏不会卡 hmdriver2（不像 iOS 锁屏会卡 lockdown），但**息屏后无法通过 web 唤起**——长时间自动化建议把"自动锁屏"调到「永不」 + 打开「一直亮屏」（设置 → 显示与亮度）
- 镜像默认走 `hypium` 后端（hypium Captures MJPEG，~30fps、<100ms）；不流畅或 hypium 不可用时切 `screenshot` 兜底，详见 [§7](#7-切镜像后端三端总表-高级可选)
- 中文输入**不需要装 ADBKeyBoard 类东西**，hmdriver2 的 `inputText` 原生支持 Unicode

### 给 UI 自动化测试团队的"raw driver 后门"

需要绕过 VLM 视觉链路、直接写控件脚本（XPath / 元素树）的同事，看 [`HarmonyOS环境配置笔记.md §6`](./HarmonyOS环境配置笔记.md#6-给-ui-自动化测试团队的后门)。**业务测试同事只用 web 浏览器操控真机，不需要装任何东西**。

## 6. WDA 7 天到期（免费 Apple ID）

症状：

- iPhone 上 `Automation Running` 消失 / 打不开
- agent 日志报 `WDA 未在 Ns 内就绪` / `Failed to load the test bundle`

## 7. 切镜像后端（三端总表，高级可选）

> **铁律**：每端都至少有一条**主路径**（默认开箱即用）+ 一条**降级路径**（主路径出问题时改 env 立刻回退），随时切。**Android / iOS / HarmonyOS 三端互不影响**，env 只动哪一端，哪一端重启 agent 生效。

### 7.1 全表速览

| 端 | env 名 | 主路径（默认） | 降级路径 |
|---|---|---|---|
| **Android** | 无（无切换；scrcpy 单路） | scrcpy → ffmpeg fmp4 → MSE，~30-60fps | — |
| **iOS** | `AI_PHONE_IOS_MIRROR_BACKEND` | **`mjpeg_passthrough`** —— WDA mjpeg 直推 `<img>`，~15-20fps、旋转自适应 | `wda_mjpeg`（同源 + ffmpeg → MSE）/ `dvt_screenshot`（无 WDA 兜底，~2-3fps、发烫） |
| **HarmonyOS** | `AI_PHONE_HARMONY_MIRROR_BACKEND` | **`hypium`** —— hypium Captures MJPEG socket 长连，~30fps、<100ms、折叠/异形屏自适应 | `screenshot`（hdc 截图轮询，~8-10fps、CPU 高，hypium 不可用时兜底） |

### 7.2 iOS 详表

| backend | 原理 | 优点 | 缺点 |
|---|---|---|---|
| **`mjpeg_passthrough`**（**默认、推荐**） | WDA mjpeg server → JPEG 原样推到浏览器 `<img>` | 每帧独立、**设备旋转天然自适应**、无 ffmpeg、延迟最低 | 需要 WDA 已就绪 |
| **`wda_mjpeg`** | 同上拉 mjpeg，但过 ffmpeg 编 H.264 → fmp4 → MSE | 画面解码走浏览器原生硬解 | MSE 的 init segment 定死分辨率，设备旋转时链路更脆弱（已知：转屏后要刷页面才修正） |
| **`dvt_screenshot`** | pmd3 DVT `Screenshot` 轮询 PNG | 不依赖 WDA | ~2-3fps、iPhone 发烫、依赖 tunneld + DDI |

```bash
# backend/.env 任选其一，重启 agent 终端 E 生效
AI_PHONE_IOS_MIRROR_BACKEND=mjpeg_passthrough    # 推荐主路径
# AI_PHONE_IOS_MIRROR_BACKEND=wda_mjpeg          # MSE / H.264 后备
# AI_PHONE_IOS_MIRROR_BACKEND=dvt_screenshot     # WDA 完全跑不起来时的兜底

# mjpeg_passthrough 与 wda_mjpeg 共用：
# AI_PHONE_WDA_MJPEG_FPS=20
# AI_PHONE_WDA_MJPEG_QUALITY=70          # passthrough 建议 70-80
# AI_PHONE_WDA_MJPEG_LONG_EDGE=720
```

> **两个 mjpeg 方案都要求 WDA 已在跑**（即 `xcodebuild test` 起来、`/status` 绿）。
> §4 的终端 A（`pmd3 tunneld`）对 mjpeg 两路**不是必须**，但建议开着——iOS run 流程里其他功能（截图诊断、lockdown 查询等）仍会用到 DVT。

### 7.3 HarmonyOS 详表

| backend | 原理 | 优点 | 缺点 |
|---|---|---|---|
| **`hypium`**（**默认、推荐**） | uitest socket 长连，发 `Captures startCaptureScreen` → 设备主动 push JPEG 帧序列；和 hmdriver2 的 RecordClient 同款协议 | ~30fps、<100ms、**每帧自带尺寸 → 折叠屏/横竖屏切换天然自适应**、agent 端零编解码 | 依赖设备端 `uitest` daemon 健康 |
| **`screenshot`** | `hdc shell snapshot_display` 截 PNG → `hdc file recv` 拉回来 → JPEG 重压 | 不依赖 hypium socket，hmdriver2 完全坏掉时还能兜 | ~8-10fps（USB2 + 编码 + hdc 往返物理上限），CPU 高、agent 日志密集 |

```bash
# backend/.env，重启 agent 终端 E 生效
AI_PHONE_HARMONY_MIRROR_BACKEND=hypium           # 推荐主路径
# AI_PHONE_HARMONY_MIRROR_BACKEND=screenshot     # 兜底

# 下面三项仅 screenshot 后端生效（hypium 走设备硬编码原图）：
# AI_PHONE_HARMONY_MIRROR_FPS=8
# AI_PHONE_HARMONY_MIRROR_JPEG_QUALITY=55
# AI_PHONE_HARMONY_MIRROR_LONG_EDGE=720
```

> 协议层完整说明（hypium 不是 H.264 而是 MJPEG 的"反高潮"真相、为什么不复用 hmdriver2 的控制 socket）见 [`HarmonyOS接入方案_2026-04-20.md` §13](./HarmonyOS接入方案_2026-04-20.md#13-p3-b-落地完成--hypium-captures-mjpeg2026-04-21)。

## 7.5 iOS 即插即用（自动化 7x24 推荐）

默认是「按需启动」：你得点 web 上的**进入工作台**才开始编 WDA，首次冷编 1~3 分钟。

如果你的场景是**自动化随时来调用**（比如定时脚本、另一个服务挂着 API），等不起这 1~3 分钟，就开即插即用：

```bash
# backend/.env 追加
AI_PHONE_IOS_WDA_PRELOAD=true        # 插上 iPhone 就后台拉 WDA（默认 false）
AI_PHONE_IOS_WAKE_ON_ENTER=true      # WDA 就绪后自动点亮屏幕（默认 true）
```

**效果**：

- 把 iPhone 插进 Mac → agent 5s 内 rescan 探测到 → 后台起线程跑 `xcodebuild test` → WDA ready
- **首页 `/` 的设备卡片**会实时显示进度（「WDA 编译中 · 60s」这种橙/蓝色小框），跟工作台里一样
- 就绪后自动调 `WDA unlock` 亮屏，Face ID 机型不会息屏把 `lockdown` 又卡死
- 自动化进程这时直接打 API 发 `start_run` 就能秒跑

**代价**：Mac 会一直有一个 `xcodebuild test` 子进程在跑（空载时 ~100MB 内存、CPU < 1%），多台 iOS 并发时记得算好资源。

> 关了这俩开关 = 回到历史按需行为，不影响已有功能。

## 8. 健康自检（一行扫全链路）

在任意一个新终端跑：

```bash
cd /Users/dongxin/代码文件/sonic合集/ai-phone/backend && source .venv/bin/activate && \
  echo "=== iOS ===" && \
  (pgrep -fl 'pymobiledevice3 remote tunneld' | head -1 || echo '❌ 终端 A tunneld 没开（仅 iOS 用）') && \
  (pymobiledevice3 usbmux list 2>/dev/null | grep -qi 'udid' && echo '✅ iPhone 在 usbmux 列表' || echo '⚠️  iPhone 没接上/没 Trust（不用 iOS 可忽略）') && \
  (curl -s --max-time 2 http://127.0.0.1:8100/status | grep -q 'ready' && echo '✅ WDA OK' || echo '⚠️  WDA 不可达（Agent 没起 or 还在编 WDA；不用 iOS 可忽略）') && \
  echo "=== Android ===" && \
  (adb devices 2>/dev/null | awk 'NR>1 && $2=="device"{n++} END{exit !n}' && echo '✅ Android 设备在线' || echo '⚠️  Android 没接上/未授权（不用 Android 可忽略）') && \
  echo "=== HarmonyOS ===" && \
  (command -v hdc >/dev/null 2>&1 && echo '✅ hdc 在 PATH' || echo '⚠️  hdc 不在 PATH（不用 HarmonyOS 可忽略；agent 内置 PATH fallback 仍可能拉到）') && \
  (hdc list targets 2>/dev/null | grep -vqE '^\s*\[Empty\]\s*$' && echo '✅ HarmonyOS 设备在线' || echo '⚠️  hdc list targets 空（不用 HarmonyOS 可忽略）') && \
  echo "=== 通用 ===" && \
  (curl -s --max-time 2 http://127.0.0.1:8000/healthz >/dev/null && echo '✅ Server OK' || echo '❌ 终端 D Server 没开') && \
  (curl -s --max-time 2 http://127.0.0.1:5180 >/dev/null && echo '✅ 前端 OK' || echo '❌ 终端 F 前端没开')
```
