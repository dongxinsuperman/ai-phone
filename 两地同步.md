# 两地同步：新 Mac 拉下仓库到能跑通

> **本文档定位**：ai-phone 仓库做的是"两台 Mac 之间的进度同步盘"。任意一台 Mac（不管是第一台还是新机）拉下仓库后，按本文档**从上到下做一遍**就能跑起来。
>
> 完整跑通的标志：浏览器进 `http://127.0.0.1:5180`，能看到设备卡片，输入 goal 能跑出 step。

## 1. 系统级软件准备

### 1.1 必装（所有人）

```bash
# 1) Homebrew
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 2) brew 一行装齐基础栈
brew install python@3.11 node ffmpeg android-platform-tools git

# 3) 验证
python3.11 --version    # Python 3.11.x
node -v                  # v18+ 或 v20+
ffmpeg -version | head -1
adb version
```


| 包                        | 用途                                     |
| ------------------------ | -------------------------------------- |
| `python@3.11`            | 后端 venv（pyproject.toml 锁了 3.11–3.12）   |
| `node`                   | 前端 vite，自带 npm                         |
| `ffmpeg`                 | Android scrcpy 镜像 + iOS wda_mjpeg 镜像必需 |
| `android-platform-tools` | adb（Android 真机）                        |
| `git`                    | clone 代码（macOS 自带也行，brew 版更新）          |


### 1.2 按需装（看你这台 Mac 接什么真机）


| 接什么             | 还要装什么                                                                                                                                           |
| --------------- | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| **只接 Android**  | 上面 §1.1 就够                                                                                                                                      |
| **接 iOS**       | App Store 装 **Xcode**（14GB+，启一次同意 license），然后：`sudo xcode-select -s /Applications/Xcode.app/Contents/Developer`                                 |
| **接 HarmonyOS** | [DevEco Studio](https://developer.huawei.com/consumer/cn/deveco-studio/)（按 Mac 芯片选 ARM 或 X86 版），拖进 `/Applications/`，**不用启动 IDE**，借里面的 hdc 二进制即可 |


iOS 用户额外做（4 步，按顺序，每步都给了"成功标志"）：

**步骤 1 — Xcode 启一次，同意 license**

App Store 装好 Xcode 后，**双击 `/Applications/Xcode.app` 启动一次**：

- 弹 "Install Components" → 点 Install → 输 Mac 密码
- 弹许可协议 → 拉到底点 Agree
- 主界面显示出来即可，可以关掉

> 不启动这一次后面 `xcodebuild` 跑会报 `License has not been agreed to`。

**步骤 2 — 把命令行工具切到完整 Xcode**

```bash
sudo xcode-select -s /Applications/Xcode.app/Contents/Developer
```

输 Mac 密码后无回显 = 成功。

> 这步是告诉系统：以后 `xcodebuild` 走 `/Applications/Xcode.app` 而不是只装了 Command Line Tools 的精简版。agent 内部就是靠 `xcode-select -p` 找 `xcodebuild`。

**步骤 3 — 立刻验证**

```bash
xcode-select -p
xcodebuild -version
```

期望输出：

```
/Applications/Xcode.app/Contents/Developer        ← 第 1 行
Xcode 26.x.x                                       ← 第 2 行（具体版本号）
Build version XXXXXX
```

> 如果第 1 行不是 `/Applications/Xcode.app/Contents/Developer`，回到步骤 2 重做。
> 如果 `xcodebuild -version` 报 license 没同意，回到步骤 1 启动 Xcode 同意一遍。

**步骤 4 — Xcode 登录 Apple ID**

打开 Xcode → 顶部菜单 **Xcode → Settings** → **Accounts** 标签 → 左下 **+** → **Apple ID** → 输你的 Apple ID（免费 ID 也行）+ 密码。

成功标志：Accounts 列表里出现你的 Apple ID，右侧 Team 列表里能看到你的 Personal Team / 公司 Team。

> 这一步把开发者私钥 + 证书写进 macOS Keychain。`xcodebuild test` 自动从 Keychain 读签名信息，所以以后 agent 自动签名能走通。

> ✅ **iOS 用户都做完上面 4 步**，本节才算完。如果你这台 Mac **以前已经装过 Xcode 并跑通过 iOS 开发**，能跑过 `xcode-select -p` + `xcodebuild -version` + Accounts 里有 Apple ID，**直接跳过即可**。

## 3. 配置 `backend/.env`（核心步骤）

这是**唯一一个本机自己写的文件**。`.env` 在 `.gitignore` 里，永远不会上传到 GitHub，两台 Mac 各自维护各自的。

### 3.1 复制模板

```bash
cd ai-phone/backend
cp .env.example .env
```

### 3.2 必填字段（5-7 个）

打开 `.env`，按下面表格修改。**其它字段保持模板默认即可**。


| 字段                           | 怎么填                                                                                                                   |
| ---------------------------- | --------------------------------------------------------------------------------------------------------------------- |
| `AI_PHONE_DB_URL`            | Postgres 连接串。共用远程同事的库 / 本地起 PG 都行（见 §3.3）                                                                             |
| `AI_PHONE_VLM_API_KEY`       | 火山引擎方舟 key（在 [console.volcengine.com/ark](https://console.volcengine.com/ark) 申请，或问另一台 Mac 的 .env 拷过来）                |
| `AI_PHONE_ASSISTANT_API_KEY` | 留空则自动复用 `AI_PHONE_VLM_API_KEY`，多数情况不用填                                                                                |
| `AI_PHONE_WDA_PROJECT_DIR`   | **绝对路径**，写本机 clone 后的位置：`/Users/<本机用户>/<clone 位置>/ai-phone/third_party/WebDriverAgent`（仅 iOS 用户必填）                    |
| `AI_PHONE_WDA_BUNDLE_ID`     | 自定义唯一值，建议 `com.<本机用户名>.wda`（避免免费 Apple ID 同 Bundle Id 撞配额；仅 iOS 用户必填）                                                 |
| `AI_PHONE_WDA_TEAM_ID`       | 你的 Apple Developer Team ID（10 字符大写，在 [developer.apple.com/account](https://developer.apple.com/account) 查；仅 iOS 用户必填） |
| `AI_PHONE_AGENT_TOKEN`       | 默认 `dev` 即可，本地开发不用改                                                                                                   |


### 3.3 关于数据库

两种选项：


| 选项                         | .env 怎么写                                                                                                                                                    | 注意                                   |
| -------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------ |
| **共用远程同事的 PG**（10.8.8.120） | 直接拷另一台 Mac 的 .env DB 串                                                                                                                                      | **同时只一台 Mac 跑 agent**，否则两边 run id 会撞 |
| **本地各起 PG**                | `brew install postgresql@15` → `brew services start postgresql@15` → `createdb auto_app` → 改 `.env` 为 `postgresql+asyncpg://<本机用户>@127.0.0.1:5432/auto_app` | 启动时代码自动建表，全新空库                       |


### 3.4 关于 WDA 签名（仅 iOS 用户）

新增的 `AI_PHONE_WDA_BUNDLE_ID` + `AI_PHONE_WDA_TEAM_ID` 会被 agent 在调 `xcodebuild` 时作为命令行 build settings 注入：

```
xcodebuild test ... PRODUCT_BUNDLE_IDENTIFIER=com.<你>.wda DEVELOPMENT_TEAM=<你的Team>
```

效果：`third_party/WebDriverAgent/.../project.pbxproj` 文件**不需要改**（即使 git 上看到的是另一个人的 Bundle Id / Team），每台 Mac 用自己 .env 注入自己的签名，**两机彻底解耦**。

## 5. iOS 用户额外步骤（首次一次性）

> **只接 Android / 鸿蒙真机的同学跳过整节。**
>
> 本节做完一次，以后这台 iPhone 在这台 Mac 上就"认识"了。日常每天起服务只走 §6。

iOS 首次接入按这 7 个步骤走，**一步一步打勾**，每步都标了"如果你已经满足这一步的状态，可以跳过"。

### 5.1 检查前置条件（30 秒）

确认下面 3 件事都做完了：

- [ ] §1.2 iOS 用户的 4 步全部 ✅（Xcode 启动过、`xcode-select` 切了、`xcodebuild -version` 过、Apple ID 登过）
- [ ] §3.2 `.env` 里 `AI_PHONE_WDA_PROJECT_DIR` / `AI_PHONE_WDA_BUNDLE_ID` / `AI_PHONE_WDA_TEAM_ID` 三个字段都填了
- [ ] §4.1 装依赖时**加了 ios extras**（`pip install -e ".[ios]"` 或 `pip install -e ".[ios,harmony]"`）

少一个回去补，否则后面命令找不到 `pymobiledevice3`、xcodebuild 找不到工程目录。

### 5.2 iPhone 首次接入这台 Mac

> ✅ **如果你这台 iPhone 之前已经在这台 Mac 上 Trust 过、并且 `pymobiledevice3 usbmux list` 能看到它**，跳到 §5.3。

按顺序：

1. **先亮屏解锁 iPhone**——这是关键。锁屏状态下插数据线，"信任此电脑"对话框**不会弹**，会让你以为是线坏了
2. 用**原装或 MFi 数据线**把 iPhone 插到 Mac（很多充电线只通电不通数据，agent 后面会刷 "udid 不在 usbmux 列表里"）
3. iPhone 上弹 **"要信任此电脑吗？"** → 点 **Trust / 信任** → 输 iPhone 锁屏密码
4. **把 iPhone 自动锁定调成"永不"或 ≥15 分钟**：iPhone 设置 → 显示与亮度 → 自动锁定 → 永不

   > 为什么硬性要求：iOS 18 / 26 起，设备锁屏时连最基本的 `lockdown StartSession` 都返回 `PasswordProtected`。agent 每 5 秒 rescan，iPhone 默认 30 秒就锁屏，一锁 agent 就读不到设备元信息，前端设备卡进入"锁屏中"橙色状态，操作全部挂起。

5. **验证 Mac 已经认到 iPhone**：

   ```bash
   cd ai-phone/backend
   source .venv/bin/activate
   pymobiledevice3 usbmux list
   ```

   期望输出（截取关键字段）：

   ```json
   [
     {
       "Identifier": "00008130-XXXXXXXXXXXXXXX",   ← 这就是 UDID（40 字符），记下来后面要用
       "DeviceName": "你的 iPhone 名字",
       "ProductType": "iPhone15,3",
       ...
     }
   ]
   ```

   **看不到任何条目就 = 失败**。处理：拔下重插 → 让 "信任此电脑" 弹窗再来一次 → 再跑这条命令。

### 5.3 iPhone 打开"开发者模式"（iOS 16+ 必须）

> ✅ **如果你的 iPhone 设置 → 隐私与安全性 → 已经能看到"开发者模式"且开关已开**，跳到 §5.4。

1. iPhone：**设置 → 隐私与安全性** → 拉到最底下找 **「开发者模式」** → 打开开关
2. iPhone 提示重启 → 重启
3. 重启后亮屏解锁 → 再进 **设置 → 隐私与安全性 → 开发者模式** → 按提示 **「打开」** → 输 iPhone 密码

   > 看不到「开发者模式」菜单怎么办：先做完 §5.5 让 agent 跑一次 `xcodebuild test`（哪怕首次失败），iOS 在第一次往设备装开发者 App 后会把这个菜单显出来，再回来打开开关。

### 5.4 挂 DDI（开发者磁盘镜像）

> ✅ **如果 iPhone 没断电过、之前已经成功挂过 DDI**，跳到 §5.5。
> ⚠️ **iPhone 每次断电 / 重启 / 系统升级，都要重做这一步**（DDI 挂载在重启后会丢）。

DDI = "iOS 开发者磁盘镜像"，iOS 17+ 上的截图、调试服务依赖它。**没挂 → agent 截图全报 `InvalidService`**。

**步骤 1：先查 UDID**

```bash
cd ai-phone/backend
source .venv/bin/activate
pymobiledevice3 usbmux list
```

复制输出里 `Identifier` 那一长串（40 字符，类似 `00008130-001A2B3C4D5E6F70`），就是你的 UDID。

> 嫌敲长 UDID 麻烦的话：把它写进 shell 变量备用，比如：
> ```bash
> export IOS_UDID=00008130-001A2B3C4D5E6F70
> ```
> 后面命令直接用 `$IOS_UDID` 即可。

**步骤 2：挂 DDI**（把上面查到的 UDID 粘到 `--udid` 后）

```bash
sudo -E .venv/bin/python -m pymobiledevice3 mounter auto-mount --udid 00008130-001A2B3C4D5E6F70
```

输 Mac 密码（首次会要 sudo）。

**成功标志**——看到下面任一行即可：

- `INFO DeveloperDiskImage mounted successfully` ✅ 第一次挂上
- `ERROR DeveloperDiskImage already mounted` ✅ 已经挂过（不是真的错，等同成功）

> 这条命令做完就关，不用留着。**iPhone 重启后回来重做一次。**

### 5.5 启动 agent，触发首次 xcodebuild test

到这一步先按 §6 把 4 个终端开起来（**iOS 用户启动顺序：A → D → E → F**）。**第一次起 Agent (终端 E) 时**：

- agent 自动跑 `xcodebuild test -allowProvisioningUpdates ...`
- 终端 E 日志会刷大量 `xcodebuild test ...` / `Building` / `Compiling`
- **首次冷编 1~3 分钟**，别 Ctrl+C，让它编下去
- 编完 agent 把 WDA app 装到 iPhone 上

如果 1 分钟内没看到任何 `xcodebuild` 日志，回头检查 §3.2 `.env` 里的 `AI_PHONE_WDA_PROJECT_DIR` 是不是写错。

### 5.6 iPhone 上点"信任开发者证书"（首次必做）

> ✅ **如果 iPhone 上已经出现 `Automation Running` 灰屏**，说明信任已 OK，跳到 §5.7 自检。
> ✅ **如果你这个 Apple ID 之前已经在这台 iPhone 上信任过、且没换过 Bundle Id / Team**，**多数情况下也跳过**——但如果终端 E 报 `Developer App Certificate is not trusted`，回到本步处理。

§5.5 触发的首次 `xcodebuild test` 编完装包后，iPhone 上会弹：

> **"无法验证 App"** —— "iPhone 上没有可用于验证此 App 开发者的互联网连接……"

这是 Apple 在做开发者证书审批，**这个对话框不点，WDA 跑不起来**。处理：

1. iPhone 上：**设置 → 通用 → VPN 与设备管理**
2. 在 "**开发者 App**" 分类下找你的 Apple ID（免费 ID 显示成 `Apple Development: xxx@xxx.com`）→ 点进去
3. 点 **「信任 "xxx@xxx.com"」** → 弹二次确认 → **「信任」**
4. 列表里那一行出现绿色"✓ 已验证" = 成功

**回到 Mac 终端 E**：

- agent 内置 45s 自动 retry，等一会儿 `xcodebuild test` 会自动重试
- 不想等：在终端 E 按 `Ctrl + C` → 重新 `python -m ai_phone agent` 起一次

### 5.7 看到 `Automation Running` 灰屏 = 大功告成

成功的最终标志（同时满足）：

- iPhone 屏幕上出现 **灰底白字 `Automation Running`**
- 终端 E 日志出现 `WDA ready` / `WDA /status OK`
- 浏览器 `http://127.0.0.1:5180` 设备卡片状态变绿、能看到 iPhone 镜像画面

> **以后什么时候还会要求重新走一遍 §5.6 的"信任开发者"**：
> - 免费 Apple ID 的 provisioning profile **7 天到期**
> - iOS **大版本升级**后旧签名作废
> - 换 `AI_PHONE_WDA_BUNDLE_ID` 或 `AI_PHONE_WDA_TEAM_ID`
> - 删 DerivedData 后强制重编
>
> 本质都是 Apple 检测到"新签名"，要求重新审批一次。处理流程和 §5.6 一模一样。

## 6. 起终端（这是日常每天都要做的步骤）

> 详细命令也可以走 [`启动终端清单.md` §9](./启动终端清单.md#9-命令参考)，本节是 §6 自包含版。

### 6.1 启动顺序总览（一定按顺序）

不同平台开几个、按什么顺序：

| 你接什么真机 | 终端数 | 顺序 |
|---|---|---|
| **只接 Android** | **3 个** | D → E → F |
| **只接 HarmonyOS** | **3 个** | D → E → F |
| **接 iOS（含 iOS + Android / iOS + 鸿蒙）** | **4 个** | **A → D → E → F** |

> **⚠️ 顺序不能乱**：D 没起 E 连不上 Server，E 没起前端 F 看不到设备卡，A 没起 iOS 截图链路会断。
>
> 每个终端开起来后**都让它常驻**，不要 Ctrl+C 关掉。

### 6.2 终端 A — pmd3 tunneld（仅 iOS 用户，sudo 常驻）

> ✅ **不接 iOS 的同学跳过本小节**，直接看 §6.3。

```bash
cd ai-phone/backend
sudo .venv/bin/pymobiledevice3 remote tunneld
```

输 Mac 密码后开始打日志。看到类似 `RemoteServiceDiscoveryService` 之类的输出 = 起来了。

**这个终端不能关**：agent 的 iOS 截图、lockdown 查询、镜像兜底都靠它。

> 一台 Mac 重启后只需要起这一次，所有 iPhone 共用同一个 tunneld。

### 6.3 终端 D — Server（FastAPI）

```bash
cd ai-phone/backend
source .venv/bin/activate
uvicorn ai_phone.server.app:app --host 0.0.0.0 --port 8000 --reload
```

**成功标志**：日志末尾出现 `Application startup complete.` + `Uvicorn running on http://0.0.0.0:8000`。

> 想验证：另开一个终端跑 `curl -s http://127.0.0.1:8000/healthz`，能正常返回就 OK。

**这个终端不能关**：前端 F 和 Agent E 都连这个。

### 6.4 终端 E — Agent（扫描设备 / 驱动 WDA）

```bash
cd ai-phone/backend
source .venv/bin/activate
python -m ai_phone agent
```

**成功标志**：

- 日志看到 `connected to server ws://127.0.0.1:8000/ws/agent`
- 几秒内扫到真机：`registered device <UDID> platform=ios/android/harmony`
- iOS 用户会接着看到 `xcodebuild test ...` 进入编译流程（见 §5.5）

**这个终端不能关**：关了所有真机会从前端设备卡上消失。

## 8. 两地协作约定

两台 Mac 都搭好以后日常协作：


| 场景             | 怎么做                                                                                   |
| -------------- | ------------------------------------------------------------------------------------- |
| **代码同步**       | 一边 push、另一边 pull。`.env` / `.venv/` / `node_modules/` / `.data/` 都在 .gitignore 里，自然不同步 |
| **数据库共用**      | 两台都连 10.8.8.120。**同时只一台跑 agent**（避免设备序列号 / run id 撞车）                                 |
| **真机切换**       | 拔下来插到另一台 Mac：iOS 必须重新弹"信任此电脑" + 重新挂 DDI；Android / Harmony 重新点 USB 调试授权                |
| **VLM key 用量** | 共用同一个 ark key 没问题，方舟控制台看 token 总开销即可                                                  |

## 9. 常见问题速查


| 现象                                                          | 原因                                       | 怎么办                                                                     |
| ----------------------------------------------------------- | ---------------------------------------- | ----------------------------------------------------------------------- |
| `pip install -e .` 报 Python 版本不支持                           | 用了系统自带老 Python（3.9）                      | venv 时显式指定：`/opt/homebrew/opt/python@3.11/bin/python3.11 -m venv .venv` |
| `npm install` 卡半小时                                          | 默认 npm 源在国外                              | `npm config set registry https://registry.npmmirror.com`                |
| 后端报 `connect failed: 10.8.8.120:5432`                       | 网络不通                                     | 连 VPN，或本机起 PG（§3.3 选项 2）                                                |
| Agent 报 `xcodebuild: error: ... project ... does not exist` | `.env` 里 `AI_PHONE_WDA_PROJECT_DIR` 路径错了 | 改成本机 clone 实际位置                                                         |
| Agent 报 `Developer App Certificate is not trusted`          | iPhone 上还没点信任                            | iPhone → 设置 → 通用 → VPN 与设备管理 → 信任开发者证书                                  |
| iOS 镜像黑屏                                                    | DDI 没挂                                   | §5.4 重新挂                                                                |
| 端口 8000 被占                                                  | 老进程没退干净                                  | `lsof -i :8000` 看 pid → `kill -9 <pid>`                                 |
